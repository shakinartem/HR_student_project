# Seed Data Guide

Primary seed script:

```powershell
cd apps/backend
. .\.venv\Scripts\Activate.ps1
python scripts/seed_data.py
```

## What the seed script creates today

### Admin

- 1 admin user
- Telegram ID comes from the first value in `ADMIN_TELEGRAM_IDS`
- fallback local ID if unset: `900001`

### HR users

- `verhoff_hr`
- `coffee_hr`
- `quest_hr`

All seeded HR profiles are `active`.

### Companies

- Verhoff
- Coffee Point
- Quest Room

### Students

- `active_student`
- `inactive_student`
- `muted_student`

Notes:

- `active_student` has an active subscription
- `inactive_student` has no active subscription
- `muted_student` has a muted profile and no active subscription

### Tariff

- `student_monthly_access`

### Vacancies

The seed script creates:

- 10 active public vacancies
- all with `approved` moderation state

### Applications

The seed script creates:

- 1 application in `sent`
- 1 application in `accepted`
- 1 application in `rejected`

## Manual augmentation for MVP checks

The current seed script is enough to start the project quickly, but a few manual states still need to be created through the UI or API for a full MVP smoke pass.

### Active HR and HR company

Already provided by the seed script:

- active HR users
- linked companies
- active vacancies

### Student without subscription

Already provided:

- `inactive_student`

### Student with active subscription

Already provided:

- `active_student`

### Application before accept

Already provided:

- the seeded `sent` application

### Application after accept

Already provided:

- the seeded `accepted` application

### Manual-review vacancy

Create manually:

1. Log in as an active HR user.
2. Create a vacancy containing a manual-review keyword such as `crypto`, `high income`, or `unknown employer`.
3. Create and mock-confirm publication payment.

Expected result:

- vacancy ends in `manual_review`
- it appears in admin moderation
- it does not appear in the public feed

### Rejected vacancy

Create manually:

1. Log in as an active HR user.
2. Create a vacancy containing a reject keyword such as `casino`, `betting`, or `escort`.
3. Create and mock-confirm publication payment.

Expected result:

- vacancy ends in `rejected`
- it does not appear in the public feed

### Payments

Create manually:

1. Log in as `active_student` or another student.
2. Create a top-up payment from the balance screen or `POST /api/payments/create`.
3. Mock-confirm the payment.
4. Optionally create an HR publication payment from HR vacancy detail and mock-confirm it.

This gives you:

- student balance payment records
- HR vacancy publication payment records
- ledger transactions

### Complaints

Create manually:

1. Open an HR application detail.
2. Trigger the HR complaint action through the backend/API path.
3. Open the admin complaints screen.

Expected result:

- a complaint record exists
- admin can update complaint status

## Local actor reference

The exact UUIDs are generated by the database. Use these stable Telegram usernames/IDs as the practical manual-check anchors:

- admin: first `ADMIN_TELEGRAM_IDS` value or fallback `900001`
- HR users: Telegram IDs `1001`, `1002`, `1003`
- students: Telegram IDs `2001`, `2002`, `2003`

## Current gap

The seed script does not yet auto-create:

- manual-review vacancies
- rejected vacancies
- payment rows
- complaints

Those states are part of the manual checklist and can be created through existing MVP flows without adding new product features.
